STEP 1
有 preview_html？最高优先
成品积木直接拼上
🧱 成品块 — 设计师已经拼好的完整积木，直接放到底板上，不用拆。
大部分原子分子组件走这条路。
STEP 1.1
HTML 里有 data-component 占位？
data-component = 底板上的预留卡槽
🔲 积木拼好了，但底板上有个卡槽标着"这里插 Button/Icon"。
流水线工人 _resolveDataComponents() 看到卡槽 → 去零件库取对应积木插上。

例：NavBar 的 preview_html 里：
<div data-component="💙 00.05_Button / Icon" style="width:40px;height:40px"></div>
↑ 这就是预留卡槽。渲染时自动填入 Button/Icon 的真实积木。
▼ 没有 preview_html？往下找
STEP 2
有 children？DSL 递归拼装
_renderDSL() 递归遍历子积木树
📐 一份跨平台拼装说明书 — 不写 CSS，只描述"竖着排、间距 8、居中"。

流水线递归执行：遇到 Wrapper（有 layout+children）→ 造底板 + 递归每块子积木；
遇到 Leaf（有 cid）→ 调 _resolveRef() 去零件库取积木。

例：Islands/Header 的 children 树：
{ layout:"column", children: [
  { cid:"💙 StatusBar", props:{Color:"White"} }, ← Leaf
  { cid:"💙 NavBar", props:{Ghost:"True"} }, ← Leaf
  { layout:"column", gap:8, children:[...] } ← 嵌套 Wrapper
]}

🧱 好比一块有嵌套卡槽的大底板：顶层竖排底板 → 插入 StatusBar 积木 → NavBar 积木 → 再嵌套一块小底板装封面和信息区。
▼ 没有 children？查旧格式 fallback
STEP 3
有 component_refs？旧格式兼容过渡期
扁平零件清单 → 遍历每块调 _resolveRef()
📦 旧版零件清单 — 没有嵌套结构，只是一个扁平列表（StatusBar、NavBar、封面…），逐块取来按 css 属性排列。

⚠️ 插件已升级为输出 children DSL，同时向下兼容输出 component_refs。
新数据走 Step 2，旧数据走这里。逐步淘汰。
▼ 连引用都没有？自动生成 fallback
STEP 4
有 _composed_preview_html？slots 拼装
📋 有零件编号 + 尺寸规格表 — slots 精确记录每块零件的编号和尺寸。
流水线拿编号去零件库取真正的积木，按规格拼好 → 出来的模型能交互。
来源：generate.py 读 slots 规格表，逐个查零件库拼成完整模型。
STEP 5
都没有？CSS 骨架
🔲 空底板 — 只知道底板多大（width/height）、横着还是竖着（flex-direction），但上面没有积木。

看到这个 = 提示设计师"该补积木了"。

🔄 递归核心 — _renderDSL(node) 与 _resolveRef(ref)

1. _renderDSL(node) — 遍历说明书
拿到一个节点：
• 有 layout + children → 它是底板，_dslToStyle() 生成样式，递归每块子积木
• 有 cid → 它是积木编号，调 _resolveRef() 去取
↩ children 里可以无限嵌套 Wrapper，形成递归树
2. _resolveRef(ref) — 零件库取积木
① 查 COMPONENTS_DATA[cid]（找不到查别名货架 COMPONENT_REF_MAP）
② props 打分匹配 — "我要白色 iPhoneX 款"，逐个比对货架上的 variant，匹配属性最多的胜出
③ 单变体 货架只有一款 → 直接拿
3. 取到 variant → 看它的类型
成品块 preview_html → 拼上，递归结束
又是底板 有 children → _renderDSL 继续拆
旧格式 component_refs → 扁平遍历取积木
↩ 直到所有节点都变成成品块（Leaf）为止

✅ 新架构 — Layout DSL（children 树）

// 插件导出，跨平台通用
{
  "layout": "column",
  "gap": 0,
  "children": [
    { "cid": "💙 StatusBar", "props": {"Color":"White"} },
    { "cid": "💙 NavBar", "props": {"Ghost":"True"} },
    { "layout": "column", "children": [...] }
  ]
}

// Web: _dslToStyle() → inline CSS
// iOS: 直接读 layout/gap/padding → UIStackView
// Android: 直接读 → LinearLayout / ConstraintLayout

⚠️ 旧架构 — css + component_refs（扁平）

// 仅 Web 可用，iOS/Android 需二次翻译
{
  "css": {
    "display":"flex",
    "flex-direction":"column",
    "gap":"0"
  },
  "component_refs": [
    { "cid": "💙 StatusBar", "props": {...} },
    { "cid": "💙 NavBar", "props": {...} }
  ]
}

// ❌ 无嵌套结构，扁平列表
// ❌ css 是 Web 专属，其他端不能直接用
// ❌ 已停止使用，仅兼容旧数据
